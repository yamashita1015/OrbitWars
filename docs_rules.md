# Orbit Wars コンペルール

## 基本情報
- **コンペ名**: Orbit Wars (Kaggle Simulation Competition)
- **締切**: 2026-06-23
- **提出形式**: submission.tar.gz（main.py + orbit_lite/）

---

## ゲーム概要

### マップ
- 宇宙空間に複数の惑星が配置されている
- 惑星は**軌道運動**しており、毎ターン位置が変わる
- 彗星（comet）も存在し、惑星として扱われるが盤外へ出ると消滅する

### 勝利条件
- 500ターン終了時点で最も多くの資源（惑星の生産力・艦隊数）を持つプレイヤーが勝利
- 相手プレイヤーをゲームから排除することでも勝利できる

---

## 対戦形式

### 1対1（2P）と4人FFA（4P Free-For-All）の2種類がある
- どちらの形式で対戦するかは**ランダム**に決まる
- エージェントは両形式に対応している必要がある

---

## ゲームメカニクス

### 惑星
- 各惑星は毎ターン一定量の艦隊を**生産**する（production rate）
- 惑星にはオーナー（自分・敵・中立）がある
- 中立惑星は艦隊を生産するが攻撃してこない

### 艦隊の移動
- 自分の惑星から任意の惑星へ艦隊を送ることができる
- 送る艦隊数・角度・出発惑星IDをアクションとして指定
- 艦隊の速度は艦隊数に依存（多いほど速い、ただし上限あり）
  - `speed = 1 + 5 * (log(n) / log(1000))^1.5`
- 惑星は**軌道運動**しているため、将来位置を予測して角度を決める必要がある

### 惑星の占領
- 敵・中立惑星に到達した艦隊数 > 現在の守備艦隊数 → 占領成功
- 占領後はその惑星が自分のものになり、生産を開始する

### 彗星
- 特殊な惑星で、盤上を横断するように移動する
- 盤外に出ると**艦隊ごと消滅**するため、送った艦隊が失われるリスクがある

## 物理定数

| 定数 | 値 | 意味 |
|---|---|---|
| BOARD_SIZE | 100.0 | ボードの一辺の長さ（座標は 0〜100） |
| CENTER | 50.0 | ボード中心座標（x=50, y=50） |
| SUN_RADIUS | 10.0 | 中央の太陽半径（この範囲内には惑星・艦隊は入れない） |
| MAX_SHIP_SPEED | 6.0 | 艦隊の最高速度（ships→∞ の極限値） |
| ROT_RADIUS_LIMIT | 50.0 | 軌道運動する惑星の判定しきい値（orbital_radius + radius < 50 なら軌道運動） |
| DEFAULT_EPISODE_STEPS | 500 | 1ゲームの最大ターン数 |

## 惑星パラメータ

各惑星は以下の7つのフィールドを持つ（`planets[i]` = `[planet_id, owner, x, y, radius, ships, production]`）：

| インデックス | フィールド名 | 型 | 説明 |
|---|---|---|---|
| 0 | planet_id | int | 惑星ID（-1はパディングスロット＝存在しない） |
| 1 | owner | int | オーナー（絶対プレイヤーインデックス、-1=中立） |
| 2 | x | float | 現在のX座標（0〜100） |
| 3 | y | float | 現在のY座標（0〜100） |
| 4 | radius | float | 惑星の半径 |
| 5 | ships | float | 現在の艦隊数 |
| 6 | production | float | 1ターンあたりの艦隊生産数（自分が所有時） |

### 軌道運動
- `orbital_radius + radius < 50` の惑星は毎ターン中心を軸に回転する
- 角速度（`angular_velocity`）はゲーム開始時に決まり、全惑星共通（デフォルト約0.03 rad/turn）
- 静止惑星（外周付近）は軌道運動しない

### 惑星数
- 1ゲームあたり **24〜52惑星**（最大スロット数64）

---

## 艦隊パラメータ

空中にある艦隊（`fleets[i]` = `[fleet_id, owner, x, y, angle, from_planet_id, ships]`）：

| インデックス | フィールド名 | 型 | 説明 |
|---|---|---|---|
| 0 | fleet_id | int | 艦隊ID（-1はパディングスロット） |
| 1 | owner | int | オーナー（絶対プレイヤーインデックス） |
| 2 | x | float | 現在のX座標 |
| 3 | y | float | 現在のY座標 |
| 4 | angle | float | 進行角度（ラジアン） |
| 5 | from_planet_id | int | 出発した惑星のID |
| 6 | ships | float | 艦隊の艦隊数 |

### 艦隊速度の公式
```
speed = 1 + (MAX_SHIP_SPEED - 1) × (log(ships) / log(1000))^1.5
      = 1 + 5 × (log(ships) / log(1000))^1.5
```

| 艦隊数 | 速度（近似） |
|---|---|
| 1 | 1.0 |
| 10 | 2.0 |
| 30 | 2.7 |
| 100 | 3.7 |
| 300 | 4.7 |
| 1000 | 6.0（最高） |

## エージェントの入力（obs）

毎ターン `agent(obs)` が呼ばれる。`obs` はdict形式：

```python
obs = {
    "player":            int,          # 自分のプレイヤーインデックス（0〜3）
    "step":              int,          # 現在のターン数（0〜499）
    "planets":           list[list],   # 現在の全惑星情報 [planet_id, owner, x, y, r, ships, prod]
    "initial_planets":   list[list],   # ゲーム開始時の惑星情報（軌道計算に使用）
    "fleets":            list[list],   # 空中の全艦隊情報 [fleet_id, owner, x, y, angle, from_id, ships]
    "angular_velocity":  float,        # 軌道の角速度（rad/turn、全惑星共通）
    "episode_steps":     int,          # ゲームの最大ターン数（通常500）
    "next_fleet_id":     int,          # 次に作られる艦隊のID
    "remainingOverageTime": float,     # 残り超過時間（タイムアウト管理用）
    # 彗星が有効な場合のみ:
    "comet_planet_ids":  list[int],    # 現在盤上にいる彗星の惑星ID一覧
    "comets":            list[dict],   # 彗星の経路情報
    # メタデータ（存在する場合）:
    "player_count":      int,          # 2 または 4
}
```

## エージェントの出力（アクション）

`agent(obs)` の戻り値は移動のリスト：

```python
moves = [
    [from_planet_id, angle, num_ships],  # 移動1
    [from_planet_id, angle, num_ships],  # 移動2
    ...
]
```

| フィールド | 型 | 説明 |
|---|---|---|
| from_planet_id | int | 出発惑星のID（自分所有の惑星のみ） |
| angle | float | 送り出す角度（ラジアン、-π〜π） |
| num_ships | int | 送る艦隊数（1以上） |

- 1ターンに複数の惑星から同時に送ることができる
- 同じ惑星から複数の方向へ同時に送ることも可能
- 送り出した艦隊は翌ターンから空中に存在し、毎ターン進行方向へ移動する

---

## 彗星

- 彗星はゲーム中5回出現（ターン50, 150, 250, 350, 450）、1回に4個
- `COMET_RADIUS = 1.0`、`COMET_PRODUCTION = 1.0`（通常惑星より小さい）
- 盤外に出ると**惑星ごと消滅**し、その時点で送った艦隊も失われる
- 彗星を占領して盤外退出まで保持しても何も得られない → **攻撃は非推奨**

---

## 1ターンの処理順序

毎ターン、以下の順番で処理が行われる。

```
【前処理】彗星のスポーン（該当ターンのみ）
  ↓
Step 0. 艦隊の出撃（全プレイヤー同時）
  ↓
Step 1. 生産（全所有惑星が艦隊を生産）
  ↓
Step 2. 艦隊の移動（連続衝突判定）
  ↓
Step 3. 惑星の軌道運動 + 彗星の移動（スウィープ判定）
  ↓
Step 4. 戦闘解決（惑星に到達した艦隊の占領判定）
  ↓
【後処理】観測データを全プレイヤーに配布 → 次ターンへ
```

---

### 各ステップの詳細

#### 【前処理】彗星スポーン
- ターン 50, 150, 250, 350, 450 の**直前**に彗星グループが生成される
- 1回に最大4個の彗星惑星が盤外から出現し、事前計算済みの経路を移動する
- 彗星の初期艦隊数は `min(rand(1-99), rand(1-99), rand(1-99), rand(1-99))` — 小さい値に偏る

#### Step 0. 艦隊の出撃
- 全プレイヤーのアクションを**同時に**処理する
- 各移動命令 `[from_planet_id, angle, num_ships]` に対して：
  - 自分の惑星かつ艦隊数が足りる場合のみ有効
  - `num_ships` を整数に切り捨て
  - 艦隊は惑星表面のすぐ外（`惑星中心 + (radius + 0.1) × 方向ベクトル`）からスタート
  - 元の惑星の艦隊数から即座に減算される

#### Step 1. 生産
- **オーナーがいる**全惑星（中立を除く）で `ships += production` が行われる
- 出撃後に生産されるため、出撃した艦隊はその分の生産を受け取れない

#### Step 2. 艦隊の移動（連続衝突判定）
- 各艦隊を `speed = 1 + 5 × (log(ships) / log(1000))^1.5` だけ進行方向へ移動
- 移動後に以下を判定（移動前→移動後の線分で連続判定）：
  - **盤外**に出た → 艦隊消滅
  - **太陽**（中心から半径10以内）を通過した → 艦隊消滅
  - **惑星**の半径内を通過した → その惑星の戦闘リストに追加

#### Step 3. 惑星の軌道運動 + 彗星移動
- 軌道惑星が新しい位置に移動し、**移動経路上に存在した艦隊**を拾う（スウィープ）
  - 艦隊が静止していても、惑星側が動いて接触すると戦闘になる
- 彗星も経路に沿って1ステップ進み、同様にスウィープ判定

#### Step 4. 戦闘解決
各惑星に到達した艦隊をまとめて処理：

```
1. プレイヤーごとに到達艦隊数を合算
2. 最多プレイヤー（A）から2番目（B）を引く
   → A - B > 0 の場合 A が生存、= 0 の場合全滅（中立化）
3. 生存艦隊 vs 惑星守備隊：
   → 生存艦隊 > 守備隊：占領（守備隊 = 生存艦隊 - 守備隊、オーナー変更）
   → 生存艦隊 ≤ 守備隊：撃退（守備隊 -= 生存艦隊）
```

**重要な特性：**
- 自分の艦隊が自分の惑星に到達した場合は**増援**（守備隊に加算）
- 3者以上が同じターンに同じ惑星に到達した場合も上記ルールで処理される
- 艦隊同士が空中でぶつかることは**ない**（すり抜ける）

## 早期終了

500ターン前でも以下の条件で終了することがある：

| モード | 終了条件 |
|---|---|
| 1v1 | `leader_score >= 2.0 × runner_up_score` が **5ターン連続**で成立 |
| 4P FFA | `leader_score >= 2.0 × runner_up_score` が **20ターン連続**で成立 |

スコア計算式：
- 1v1: `score = 5 × production + 1 × (惑星艦隊数 + 飛行中艦隊数)`
- 4P: `score = production`（生産力のみ）
